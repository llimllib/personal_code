  • Cool Programming Metaphor

    This site gives a Python recipe for Future Threads. It's just a wrapper around a thread, but it makes threading mind-numbingly simple. To create and start a new thread running function "func", just do

    F = Future(func, arg1, arg2, ...)
    Then, when you need the result of the thread, instead of doing a join, you simply do
    result = F()
    If the thread isn't done, the calling process will block, just like a join() statement. Nothing too fancy, but it's a nice way to think about it.
